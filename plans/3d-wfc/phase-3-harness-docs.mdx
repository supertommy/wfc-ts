- [ ] Task 0.1: Create `harness/validate-3d.ts` with 3D adjacency checker `(1pt)`
  ```typescript
  // harness/validate-3d.ts
  import type { TileRule3D } from '../src/types.js';

  interface ValidationResult3D {
    valid: boolean;
    violations: number;
    checkedPairs: number;
    firstViolation?: {
      cell1: { x: number; y: number; z: number; tile: number };
      cell2: { x: number; y: number; z: number; tile: number };
      direction: string;
    };
  }

  const DX = [-1, 1, 0, 0, 0, 0];
  const DY = [0, 0, 1, -1, 0, 0];
  const DZ = [0, 0, 0, 0, -1, 1];
  const DIR_NAMES = ['left', 'right', 'up', 'down', 'front', 'back'];

  export function validate3D(
    result: Int32Array,
    width: number,
    height: number,
    depth: number,
    rules: TileRule3D[],
    periodic: boolean
  ): ValidationResult3D {
    // Build allowed adjacency sets from rules
    const allowed: Set<number>[][] = [];
    for (let d = 0; d < 6; d++) {
      allowed[d] = [];
      for (const rule of rules) {
        const dirKey = DIR_NAMES[d] as keyof TileRule3D;
        if (Array.isArray(rule[dirKey])) {
          allowed[d][rule.forTile] = new Set(rule[dirKey] as number[]);
        }
      }
    }

    let violations = 0;
    let checkedPairs = 0;
    let firstViolation: ValidationResult3D['firstViolation'];

    for (let z = 0; z < depth; z++) {
      for (let y = 0; y < height; y++) {
        for (let x = 0; x < width; x++) {
          const i = x + y * width + z * width * height;
          const t1 = result[i];
          if (t1 < 0) continue; // unresolved

          for (let d = 0; d < 6; d++) {
            let x2 = x + DX[d];
            let y2 = y + DY[d];
            let z2 = z + DZ[d];

            if (periodic) {
              x2 = (x2 + width) % width;
              y2 = (y2 + height) % height;
              z2 = (z2 + depth) % depth;
            } else {
              if (x2 < 0 || x2 >= width || y2 < 0 || y2 >= height || z2 < 0 || z2 >= depth) {
                continue;
              }
            }

            const j = x2 + y2 * width + z2 * width * height;
            const t2 = result[j];
            if (t2 < 0) continue;

            checkedPairs++;
            const allowedSet = allowed[d][t1];
            if (!allowedSet || !allowedSet.has(t2)) {
              violations++;
              if (!firstViolation) {
                firstViolation = {
                  cell1: { x, y, z, tile: t1 },
                  cell2: { x: x2, y: y2, z: z2, tile: t2 },
                  direction: DIR_NAMES[d]
                };
              }
            }
          }
        }
      }
    }

    return {
      valid: violations === 0,
      violations,
      checkedPairs,
      firstViolation
    };
  }
  ```

- [ ] Task 1.2: Create `docs/3d-wfc.md` with 3D concepts `(1pt)`
  ```markdown
  # 3D Wave Function Collapse

  ## How 3D Differs from 2D

  - 6 directions instead of 4 (left, right, up, down, front, back)
  - Grid size is width × height × depth (can be 32× larger for same linear dimension)
  - More constraints per cell means higher contradiction rates
  - Visualization is harder (can't see inside the volume)

  ## Tileset Design for 3D

  Each tile needs 6 face definitions. Common patterns:

  ### Symmetric Tiles
  A "solid block" tile might allow any solid neighbor:
  \`\`\`typescript
  { forTile: SOLID, left: [SOLID], right: [SOLID], up: [SOLID], down: [SOLID], front: [SOLID], back: [SOLID] }
  \`\`\`

  ### Directional Tiles
  A "pipe" tile only connects along one axis:
  \`\`\`typescript
  { forTile: X_PIPE, left: [X_PIPE, JUNCTION], right: [X_PIPE, JUNCTION], up: [EMPTY], down: [EMPTY], front: [EMPTY], back: [EMPTY] }
  \`\`\`

  ### Boundary Tiles
  For non-periodic grids, tiles at boundaries have fewer neighbors.
  Tiles with empty adjacency lists in some directions can only exist at those boundaries.

  ## Performance Considerations

  - 32³ = 32,768 cells vs 32² = 1,024 cells (32× more)
  - Memory usage scales with cell count × tile count × 6 directions
  - Contradiction rates may be higher; consider increasing restart budget
  - Use `'mrv'` heuristic (default) for best performance
  ```

  In 2D, each cell has 4 neighbors. In 3D, each cell has 6:

  ## 2. Direction Convention

  wfc-ts uses compass-style naming for 3D directions:

  | Direction | Axis | Index |
  |-----------|------|-------|
  | left | -X | 0 |
  | right | +X | 1 |
  | up | +Y | 2 |
  | down | -Y | 3 |
  | front | -Z | 4 |
  | back | +Z | 5 |

  ## 3. Defining 3D Rules

  Each rule specifies which tiles can be adjacent in each of 6 directions:

  \`\`\`typescript
  const rule: TileRule3D = {
    forTile: 0,
    left: [0, 1],    // tiles 0 or 1 can be to the left
    right: [0, 1],
    up: [0],         // only tile 0 can be above
    down: [0],
    front: [0, 1],
    back: [0, 1],
  };
  \`\`\`

  ## 4. Grid Indexing

  3D grids are flattened row-major, Z-outer:

  \`\`\`typescript
  const index = x + y * width + z * width * height;

  // To get coordinates from index:
  const x = index % width;
  const y = Math.floor(index / width) % height;
  const z = Math.floor(index / (width * height));
  \`\`\`
