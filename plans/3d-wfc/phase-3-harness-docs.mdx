- [x] Task 1.2: Create `docs/3d-wfc.md` with 3D concepts `(1pt)`
  ```markdown
  # 3D Wave Function Collapse

  ## How 3D Differs from 2D

  - 6 directions instead of 4 (left, right, up, down, front, back)
  - Grid size is width × height × depth (can be 32× larger for same linear dimension)
  - More constraints per cell means higher contradiction rates
  - Visualization is harder (can't see inside the volume)

  ## Tileset Design for 3D

  Each tile needs 6 face definitions. Common patterns:

  ### Symmetric Tiles
  A "solid block" tile might allow any solid neighbor:
  \`\`\`typescript
  { forTile: SOLID, left: [SOLID], right: [SOLID], up: [SOLID], down: [SOLID], front: [SOLID], back: [SOLID] }
  \`\`\`

  ### Directional Tiles
  A "pipe" tile only connects along one axis:
  \`\`\`typescript
  { forTile: X_PIPE, left: [X_PIPE, JUNCTION], right: [X_PIPE, JUNCTION], up: [EMPTY], down: [EMPTY], front: [EMPTY], back: [EMPTY] }
  \`\`\`

  ### Boundary Tiles
  For non-periodic grids, tiles at boundaries have fewer neighbors.
  Tiles with empty adjacency lists in some directions can only exist at those boundaries.

  ## Performance Considerations

  - 32³ = 32,768 cells vs 32² = 1,024 cells (32× more)
  - Memory usage scales with cell count × tile count × 6 directions
  - Contradiction rates may be higher; consider increasing restart budget
  - Use `'mrv'` heuristic (default) for best performance
  ```

  In 2D, each cell has 4 neighbors. In 3D, each cell has 6:

  {/* Interactive diagram showing 2D vs 3D neighborhood */}

  ## 2. Direction Convention

  wfc-ts uses compass-style naming for 3D directions:

  | Direction | Axis | Index |
  |-----------|------|-------|
  | left | -X | 0 |
  | right | +X | 1 |
  | up | +Y | 2 |
  | down | -Y | 3 |
  | front | -Z | 4 |
  | back | +Z | 5 |

  ## 3. Defining 3D Rules

  Each rule specifies which tiles can be adjacent in each of 6 directions:

  \`\`\`typescript
  const rule: TileRule3D = {
    forTile: 0,
    left: [0, 1],    // tiles 0 or 1 can be to the left
    right: [0, 1],
    up: [0],         // only tile 0 can be above
    down: [0],
    front: [0, 1],
    back: [0, 1],
  };
  \`\`\`

  ## 4. Grid Indexing

  3D grids are flattened row-major, Z-outer:

  \`\`\`typescript
  const index = x + y * width + z * width * height;

  // To get coordinates from index:
  const x = index % width;
  const y = Math.floor(index / width) % height;
  const z = Math.floor(index / (width * height));
  \`\`\`

  ## 5. Try It: Simple 3D Solve

  ### Pipes
  Tiles that connect along axes, like plumbing or wiring.

  ### Terrain Layers
  Foundation at bottom, walls in middle, roof at top.

  ### Voxel Buildings
