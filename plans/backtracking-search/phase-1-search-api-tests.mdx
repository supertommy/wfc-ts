# Phase 1: Search API and Red Tests

> Part of [Backtracking Search](./README.html) · **7 story points** (≤ 8)

## Goal

Add the public opt-in search API and the failing verification harness before engine behavior changes. After this phase, the package still uses restart-only search by default, all existing tests pass, and new backtracking tests are present and intentionally fail where implementation is missing.

## Skills to Reference
- `testing-gotchas` — use structural tests for API shape and behavior, not noisy runtime thresholds.
- `game-dev-gotchas` — preserve deterministic RNG draw order in tests.
- `data-parsing-gotchas` — use asymmetric socket fixtures so direction/order bugs cannot pass accidentally.
