# Phase 3: Rich 3D Pipe Network Visualizer

> Part of [Backtracking Search](./README.html) · **7 story points** (≤ 8)

## Goal

Replace the current straights-only visualizer workaround with a rich 3D pipe tileset and socket-driven renderer. After this phase, a human can see pipes turn, climb, branch, and connect through elbows, tees, and junctions at `http://localhost:3457/`.

## Skills to Reference
- `threejs-gotchas` — material, InstancedMesh, geometry orientation, color, and camera pitfalls.
- `game-dev-gotchas` — zero-allocation render/update loop and deterministic visual smoke checks.
- `impeccable` — UI clarity and explanatory controls for the visualizer.
- `data-parsing-gotchas` — asymmetric socket tests to catch left/right/up/down/front/back transposition.

### Sprint 1: Rich socket tileset in server
- [ ] Task 1.1: Replace `buildPipeTiles()` in `viz3d/server.ts:45` with a rich socket family `(2pt)`:

```ts
function buildPipeTiles(): PipeTile[] {
  const tiles: PipeTile[] = [];
  tiles.push({ name: 'empty', open: faces(), weight: 1.2, color: EMPTY_COLOR });
  tiles.push({ name: 'straight-X', open: faces(0, 1), weight: 1.4, color: PIPE_X_COLOR });
  tiles.push({ name: 'straight-Y', open: faces(2, 3), weight: 1.4, color: PIPE_Y_COLOR });
  tiles.push({ name: 'straight-Z', open: faces(4, 5), weight: 1.4, color: PIPE_Z_COLOR });
  // elbows: exactly two non-opposite openings
  // tees: exactly three openings, including at least one turn
  // junction-6: all six openings
  return tiles;
}
```

  - Exact tile names must include prefixes `elbow-`, `tee-`, and `junction-6` so tests can detect them.
  - DO NOT add cap tiles in this phase; caps reintroduce dead-end stubs and make the visual less clear.
- [ ] Task 1.2: Keep `buildRulesFromTiles()` at `viz3d/server.ts:54` as the only rule generator `(1pt)`.
  - DO NOT hand-author adjacency arrays; socket generation prevents asymmetry bugs.
