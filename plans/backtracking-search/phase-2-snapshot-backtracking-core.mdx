# Phase 2: Snapshot Backtracking Core

> Part of [Backtracking Search](./README.html) · **8 story points** (≤ 8)

## Goal

Implement opt-in, bounded checkpoint backtracking inside `WFCEngine`. After this phase, the new red tests pass, existing default restart behavior remains unchanged, and both 2D and 3D callers can opt into stronger search via `search: { strategy: 'backtrack' }`.

## Skills to Reference
- `game-dev-gotchas` — deterministic decision order, RNG-result fidelity, and zero-allocation awareness.
- `testing-gotchas` — prove the backtracking path is exercised before gating on counters.
- `software-design-patterns` — Command/undo framing for decision checkpoints.
- `lets-build-verify` — use if checkpoint restore invariants become subtle during implementation.

## Knowledge to Reference
- `qmd://tommyato-knowledge/investigations/gpu-frontier-data-structures-for-wfc.md` — propagation is a frontier/worklist; invariants must stay consistent.
- `qmd://tommyato-knowledge/references/books/algorithms-in-a-nutshell/distilled/07-path-finding-in-ai.md` — DFS/backtracking stores decisions and restores state.

## Verification Strategy

unit-tests + script smoke. The correctness bar is state restoration and deterministic success, not speed yet.

### Sprint 3: Backtracking control flow in run and stepRun
- [ ] Task 3.1: Add `private chooseTileOrder(node: number, random: Random): number[]` `(1pt)`.
  - First element must be the same tile `weightedPick(this.distribution, random.nextDouble())` would choose.
  - Remaining elements must be all other currently-possible tiles sorted by descending distribution score, tie-breaking by lower tile index.
  - This preserves current first-choice behavior and gives deterministic fallback choices.
- [ ] Task 3.2: Implement `private backtrackAndRetry(random: Random): boolean` `(2pt)`.
  - Pop exhausted checkpoints until one has a remaining choice.
  - Restore that checkpoint.
  - Apply the next choice with `applyChoice(node, chosenTile)` and then `propagate()`.
  - Increment `backtrackCount` each time a checkpoint restore is attempted.
  - Return `false` if no checkpoint can produce a valid next choice.
- [ ] Task 3.3: Modify `run()` at `src/engine.ts:342` so only `searchStrategy === 'backtrack'` saves checkpoints and calls `backtrackAndRetry` on contradiction `(2pt)`.
  - Restart mode must keep the old contradiction branch: `contradicted = true; break;`.
  - Stop backtracking when `backtrackCount >= maxBacktracks` and fall through to outer restart.
- [ ] Task 3.4: Modify `stepRun()` at `src/engine.ts:386` with the same behavior and include `backtracks: this.backtrackCount` in yielded statuses `(1pt)`.
  - DO NOT change `attempt`; it still means outer restart attempt.
